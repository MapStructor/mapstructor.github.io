What is an "orphan" layer?
An orphan is a dataset that belongs to no map — nothing links to it, so it just sits in the database taking up space. They're almost always safe to delete.

How they happen:
  • You deleted the map but the layer stayed. Deleting a map removes the map and its links, but the underlying dataset rows can remain — now unattached.
  • Import / draw that was never finished — a layer row got created, then the tab closed or the action was undone, leaving an empty or half-filled layer on no map.
  • Leftovers from operations — an "outline" companion layer, an old auto-generated copy, or a failed duplicate/merge that created the layer before erroring.
  • Tests — throwaway layers (e.g. "testline", "poly3") made while trying things, never placed on a real map.
Example: you had "US Railways v1", replaced it with a fresh import, and deleted the old map — its "Historical_Railroads (copy)" layer is now an orphan.
